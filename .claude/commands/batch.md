# BATCH AGENT — PARALLEL MULTI-SITE PIPELINE

You receive a list of business URLs and execute the complete pipeline for each one,
in parallel, as completely independent processes.

**Input:** `$ARGUMENTS` — list of URLs separated by commas or line breaks

Example:
```
/batch https://site1.com, https://site2.com, https://site3.com
```

---

## FLOW FOR EACH URL

Each URL goes through this complete pipeline, in order:

```
/scraping <url>
    ↓
/auditoria scraping/outputs/<business>-scraping.json
    ↓
/web auditoria/outputs/<business>-auditoria.json
    ↓
/web-review web/<business>/
    ↓
(if review ≠ A) /web-refine web/<business>/ | <instructions from review>
    ↓
/deploy web/<business>/ <business>
```

---

## PARALLEL EXECUTION

**CRITICAL: Each site MUST run as an independent agent using the Agent tool.**

For each URL in the list:

1. **Parse the URLs** — separate by commas, spaces, or line breaks. Trim whitespace.
2. **Launch one Agent per URL** — use the Agent tool with these parameters:
   - `description`: "Full pipeline: <domain-name>"
   - `prompt`: The full pipeline prompt (see template below)
   - Launch ALL agents in a single call (parallel tool calls)

### Prompt template for each Agent:

```
Execute the complete pipeline for this business. Work in the directory /Users/et59866/agencia/Agency-dev/

IMPORTANT: Each step must run sequentially. Do not advance to the next step until the previous one is complete.

## Step 1: Scraping
Execute the /scraping skill with the URL: <URL>
Wait for the file to be saved in scraping/outputs/

## Step 2: Audit
Identify the generated scraping file in scraping/outputs/ (the most recent one or the one matching the business).
Execute the /auditoria skill with the path to the scraping.json

## Step 3: Web
Execute the /web skill with the path to the generated auditoria.json.
This will build the complete site in web/<business>/

## Step 4: Review
Execute the /web-review skill with the path web/<business>/
Read the review output.

## Step 5: Refine (conditional)
If the review did NOT give quality "A", execute the /web-refine skill with the instructions generated by the review.
If it gave "A", skip this step.

## Step 6: Deploy
Execute the /deploy skill with: web/<business>/ <business>
Capture the production URL.

## Final output
When finished, respond EXACTLY with this format:
PIPELINE COMPLETED: <business-name> → <deploy-url>
```

---

## MONITORING AND FINAL RESULT

After launching all agents:

1. **Wait** for all agents to complete
2. **Collect** the results from each one
3. **Generate the summary table** with this format:

```
BATCH COMPLETED: X/Y sites deployed

| Business | Deploy URL |
|---|---|
| business-1 | https://business-1.vercel.app |
| business-2 | https://business-2.vercel.app |
| ... | ... |

Failed (if any):
- business-X: [failure reason]
```

---

## RULES

- **Total isolation** — each Agent works on its site without knowledge of the others
- **Do not mix contexts** — never pass data from one site to another
- **Fault tolerance** — if one site fails, the others continue. Report the failure at the end.
- **Maximum 10 URLs** per batch — if there are more, reject with: "BATCH: maximum 10 URLs per execution"
- **No intermediate confirmations** — the pipeline runs from start to finish without requesting input
- The creative brief (/web-brief) must run without waiting for user confirmation — automatically choose the tone that best applies based on the sector
- Zero extra conversation — only the final table
